Core Design Principles

Gen1’s ETD leakage forced a complete restart. Gen2’s core question became:

Under strict pre-departure leakage control, which features truly have predictive power?

Tip

Core Finding (across 12 versions): Only genuinely new external information can improve AUC; same-source re-encoding is universally ineffective.

Design

V1 includes three feature categories with strict temporal splitting (but retains ETD for comparison):

  • Schedule + Calendar: Departure month, day of week, time period, route
  • Tail Chain: Previous flight delay (simple one-hop)
  • DFW METAR Weather: Real-time weather observation at T-2h
  • ETD Snapshot (T-2h+ETD version): Airline operations system’s estimated departure time deviation

Literature references: Zhou (2025) AUC=0.898, Berkeley (2025) F1=62.9%, Chen (2019) Tail chain, Qu (2023) CNN-MLSTM

Results

V1 Best (T-2h+ETD + XGBoost): AUC = 0.7623, F1 = 0.5031

Feature Set Best AUC
Clean (schedule + calendar) 0.6226
T-24h (+ tail chain + weather) ~0.70
T-2h (+ real-time weather) ~0.72
T-2h+ETD (+ operations snapshot) 0.7623
Warning

V1’s ETD Problem: The high AUC of the T-2h+ETD version comes from ETD deviation directly reflecting delay status. Same fatal leakage as Gen1. ETD removed from V2 onward.

Key Conclusions

  • XGBoost outperforms RF and DNN across all feature versions
  • Tail chain (previous flight delay) contributed the largest AUC improvement in T-24h version
  • Deep learning (DNN) does not outperform tree-based models on tabular data

Why V2 is an Important Milestone

This is the only case where “same-source but literature-validated feature engineering” produced a significant improvement. Subsequent versions repeatedly confirmed: only genuinely new external information can break the AUC ceiling. V2’s 9 features are a borderline case (manually designed features with strong literature support).

V3: Two-Stage Departure Delay Prediction

Design: Stage I uses XGBoost regressor to predict departure delay, Stage II adds the prediction as a feature for arrival delay prediction.

Warning

Stage I R² only 0.05–0.18, insufficient prediction quality to support Stage II

Stage I root cause: Most departure delay variance comes from events after T-2h (ground delays, ATC directives, sudden weather), which are unknowable at T-2h.

V3 best AUC = 0.7357 (T-2h+Enhanced + RF), only +0.0005 above V2, practically meaningless.

First pattern confirmation: Same-source meta-features are ineffective.

V4: Destination Airport METAR Weather (11 features)

Design: Added NOAA IEM ASOS/METAR observations from 277 destination airports (T-2h).

New features: dest_wx_temp_f, wind_speed_kt, wind_gust_kt, visibility_mi, precip_1hr_in, ceiling_ft, has_thunder, has_fog, has_snow, pressure_hpa, wx_severity

V4 Best AUC = 0.7373 (T-2h+Enhanced + XGBoost), vs V2’s 0.7352, +0.0021

Positive but marginal. Reason: T-2h destination weather snapshot is a single time-point observation, unable to capture in-flight weather dynamics.

V5: BTS Monthly Airport Statistics (6 features)

Design: Added BTS destination airport monthly statistics (delay rate, avg delay, delay std dev, for both origin/dest).

Warning

V5 Best AUC = 0.7355 (−0.0018 vs V4) — negative result

Monthly granularity is too coarse for individual flight prediction; dest_airport_encoded already implicitly captures airport-level delay tendencies. Coverage: 204/286 destination airports (71.3%), 724,660 flights (90.1%).

Second pattern confirmation: Coarse-grained same-source information is ineffective.

Design

V6 builds on V4 (then-best AUC=0.7373) with two optimization steps:

Step 1: SHAP Backward Elimination

Removed 14 low-contribution features from T-2h+Enhanced, including: dfw_avg_aadt, wx_has_snow, dest_wx_has_snow, prev_flight_landed, dest_wx_has_fog, dest_wx_has_thunder, is_weekend, dest_wx_visibility_mi, dest_wx_wind_gust_kt, prev_flight_delayed_15, wx_has_fog, dest_is_international, wx_visibility_mi, is_holiday

Step 2: Optuna Bayesian HPO (100 trials)

Optuna Best Val AUC = 0.7541 | Search time: 2310 seconds (38.5 minutes)

Best hyperparameters (XGBoost): learning_rate = 0.006, max_depth = 10, gamma = 4.79, colsample_bytree = 0.33

Results

Tip

V6 Best AUC = 0.7420 (T-2h+Enhanced + XGBoost), vs V4’s 0.7373, +0.0047

Literature: Borboudakis & Tsamardinos (arXiv:2410.06815) — SHAP backward elimination; Akiba et al. (2019) Optuna; Bergstra et al. (2011) HPO

V6 became the baseline for all subsequent versions.

V7: Temporal + Route Features (8 new features)

Added: dest_delay_rate_7d/30d, route_daily_freq, aircraft_legs_today, dfw_delay_rate_1h/6h

Warning

V7 Best AUC = 0.7366 (T-2h+Enhanced + XGB+CatBoost), vs V6’s 0.7420, −0.0054

Worst single-version regression across 12 versions.

Root cause: XGBoost with depth=10 already captures these temporal patterns automatically; all 8 new features are redundant. V6’s colsample_bytree=0.33 was not re-optimized for new features.

V8: SHAP Interaction Features + Focal Loss

Two attempts: 1. SHAP interaction features (15, e.g., dep_hour × wx_severity) 2. Focal Loss for class imbalance (positive rate 23.3%)

Warning

V8 Best AUC = 0.7407, both negative

SHAP interactions: depth=10 already implicitly captures interaction effects; manual addition is redundant. Focal Loss: At 23.3% imbalance, it trades AUC for F1 (F1 improved to 0.4744, but AUC decreased).

V7 and V8 consecutive negative results confirm: AUC ≈ 0.74 is the current feature space ceiling. Entirely new external information is needed to break through.

Design

V9 introduces TAF (Terminal Aerodrome Forecast) weather forecast data — fundamentally different from V4 METAR “observations”: TAF forecasts “future weather conditions at arrival time.”

Data source: Iowa Environmental Mesonet (IEM) public archive, 212 airports (covering 89.9% of destinations)

Tip

V9 Best AUC = 0.7440 (T-2h+Enhanced + XGBoost), vs V6’s 0.7420, +0.0020

Broke the V7–V8 negative streak.

Why TAF Works but METAR is Limited

TAF is a forecast system output of “future weather prediction,” reflecting expected weather conditions at the arrival time window even at T-2h prediction time; METAR is only “current weather observation,” with limited representativeness for destination weather on long-haul flights (flight time > 2 hours).

Pattern confirmed again: Only genuinely new external information (meteorological forecast system output) can break the AUC ceiling.

Literature: Rebollo & Balakrishnan (2014); Kim et al. (2016)

V10: Target Encoding + Network Weather Proxy

Target Encoding: Replace dest_airport, gate, subfleet label encoding with sklearn TargetEncoder(cv=5).

Warning

Negative reason: XGBoost depth=10 is already effective enough with label encoding; TE advantages are mainly seen in linear models and shallow trees (Pargent et al. 2022).

Network Weather Proxy: Average weather severity across 9 major Hub airports (ATL, ORD, DEN, LAX, CLT, JFK, SFO, EWR, LAS).

Warning

V10 Best AUC = 0.7429 (−0.0011 vs V9)

Network proxy reason: Existing DFW congestion features already capture local delay status; remote Hub weather impact on DFW is too weak within the T-2h window.

V11: NOTEARS DAG + 3-Model OOF Stacking

NOTEARS DAG: Use continuous optimization to learn causal graphs, add features from same causal clusters to XGBoost’s interaction_constraints.

Warning

Fatal issue: Linear NOTEARS found 0 edges from any feature to delayed_15.

Markov Blanket is empty; interaction_constraints severely restrict XGBoost’s learning ability. XGB_DAG AUC plummeted to 0.6793 in T-2h+Enhanced (vs V9’s 0.7440, a 0.065 gap).

3-Model OOF Stacking: XGBoost + CatBoost + LightGBM, logistic regression meta-learner.

Warning

V11 Best AUC = 0.7414 (−0.0026 vs V9)

Three GBDT models are too correlated; Stacking cannot benefit from complementarity. LightGBM received negative weight in the meta-learner.

V12: Delay-Absorption Meta-Feature

Inspiration: arXiv:2512.08197 (Li 2025, AUC=0.898, but relies on departure delay features)

Design: Stage I CatBoost predicts “whether previous delay is absorbed” (absorb_label), Stage II adds absorb_score.

Stage I OOF AUC = 0.6946 (absorption label absorb=1 accounts for 69.5%)

Warning

V12 Best AUC = 0.7423 (−0.0017 vs V9)

Stage I quality insufficient; 8 input features (turnaround_min, etc.) already exist directly in V9 feature set — Stage I is just nonlinear reorganization, not new information. Li (2025)’s high AUC relies on departure delay features (leakage); our strict version cannot replicate.

V9 confirmed as the best model across all 12 versions (AUC = 0.7440).

Issue Discovery

V9’s tail chain uses scheduled turnaround thresholds to determine “whether the previous flight has arrived” — but this judgment has a subtle leakage risk: at prediction time (T-2h), some previous flights haven’t actually arrived but are incorrectly marked as arrived.

Fix

Changed to using actual arrival time instead of scheduled time to determine tail chain availability.

Impact Scope Proportion
T-2h version affected flights 1.04%
T-24h version affected flights 77.9% (→ T-24h version permanently deprecated)

V9c Final AUC = 0.7418 (−0.0022 vs V9, only 1.04% flights affected)

V9c became Gen2’s official finalized version, the foundation for all subsequent analyses (Phases 1–4).

V9d: Right-Censored Tail Chain (Research Use)

V9d further changes tail chain from “previous arrived → use actual delay, not arrived → use 0” to right-censoring, using 7 features to encode “known partial information when previous flight hasn’t arrived” (overtime minutes, estimated remaining time, etc.), NaN for “unknowable at this time.”

V9d Temporal AUC = 0.7291 (vs V9c 0.7418, −0.0127)

Random split AUC = 0.7760 (+0.047), the gap reconfirms the importance of temporal splitting.

V9d is used for Phase 1–4 comprehensive analysis (43 features, including 7 right-censored tail chain features).
