動機： Gen2 使用 AA 私有 Snowflake 資料，無法公開重現。需要在公開資料上驗證方法論。

資料來源： BTS（Bureau of Transportation Statistics）公開資料

規模： 19,150,000 美國國內航班（2021–2023）

Temporal split：

Set Period Rows
Train 2021–2022 ~12.4M
Val 2023 H1 ~3.3M
Test 2023 H2 ~3.5M

核心差異：

  • Gen2：單一航空公司（AA）單一 Hub（DFW），804K 航班
  • Gen3：全美所有航空公司、所有機場，19.15M 航班

兩個版本：

內容： 完整復刻 Gen2 方法論 — SHAP feature selection + Optuna HPO + XGBoost

核心目的： 建立最嚴格的 leakage-free baseline — 移除所有需要 aircraft identity 或 aggregation 的特徵。

12-Rule Leakage Control Framework

# Rule V4 Compliance
1 T_pred = CRS_DEP_TIME - 2h PASS
2 Temporal gating: all features <= T_pred PASS
3 Aircraft identity: no TAIL_NUM, no tail chain PASS
4 Actual outcomes: DEP_TIME/ARR_TIME removed PASS
5 Aggregation: no congestion (baseline) PASS
6 Weather timestamps: METAR backward, TAF issue < T_pred PASS
7 Network propagation: not used PASS
8 Label leakage: ARR_DELAY -> binary, then dropped PASS
9 Dataset split: temporal only PASS
10 Global statistics: not used PASS
11 Feature verification: per-feature audit log PASS
12 Leakage audit: assertion pass required PASS

V2 vs V4 差異

V4 比 V2 少了 0.0670 AUC，量化了以下四項的綜合貢獻：

  1. Tail chain features（6 features）
  2. Congestion features（3 features）
  3. Optuna HPO
  4. SHAP feature selection

目的： 在相同 BTS 資料上比較 GBDT 與 Deep Learning 模型。

GBDT 全面優於 DL

XGBoost AUC = 0.687 >> Best DL AUC = 0.623（ΔAUC = +0.064）

所有 4 個 GBDT 模型均優於全部 7 個 DL 模型，與 Gen2 Phase 2 結論一致。

Key Insight

無論是否包含 tail chain，Schedule 都是最重要的 source group。V2 中 tail chain 壓制了 schedule 的表面佔比，但 V4 揭示了 schedule 的真實主導地位。

核心發現：看排班就知道會不會延誤

在 V4 Clean Baseline 中，schedule-derived features 佔 SHAP 總量的 59.4%。延誤的可預測部分，大部分在排班表確定的那一刻就已決定。

白話解釋： 延誤不是隨機意外，而是系統性的結構問題。航班的出發時間、使用的機型、航線的繁忙程度——這些在排班表公布時就已經決定的因素，主導了延誤的可預測性。

兩個版本的一致性：

  • V2（含 tail chain）：Schedule 18.6%，但 dep_hour 仍是 #1 individual feature
  • V4（無 tail chain）：Schedule 59.4%，結構性主導更加明顯

文獻對比：

  • FDPP-ML (2023, Journal of Big Data)：曾說「目前沒有研究能只靠 schedule 特徵預測延誤」然後自己做到了
  • 但沒有論文用 SHAP 量化並解釋「schedule 主導的比例是多少、為什麼」
  • 在全美 19.15M 航班的規模上嚴格驗證，不是地理特例
修正：Weather 並非「近零」

V4 Clean Baseline SHAP 結果：Origin METAR 11.7% + Dest METAR 8.0% + TAF 6.8% = 合計 26.5%

V2 vs V4 的天氣貢獻差異：

Version Weather Total 原因
V2 ~23.9% 被 tail chain（35.5%）壓制，表面佔比低
V4 26.5% 無 tail chain，天氣的真實貢獻更清晰

修正後的結論：

  • 天氣在嚴格 leakage control 下仍佔 26.5%，不是「近零」
  • Gen2 V9 中 TAF 帶來 +0.002 AUC 也證實天氣有正向貢獻
  • 先前「天氣近零」的說法可能是 V2 中天氣被 tail chain 壓制的假象

文獻對比：

  • Hatipoglu & Tosun (2024, MDPI)：發現天氣不重要，歸因於土耳其氣候穩定
  • Alfarhood et al. (2024, Wiley)：沙烏地阿拉伯天氣不重要，歸因於沙漠氣候
  • 我們的發現：在嚴格 leakage control 下，天氣仍佔約四分之一的 SHAP 貢獻
文獻啟示

現有多數文獻的 weather feature importance 可能因 leakage 而被高估。但在嚴格條件下，天氣仍是有意義的預測因子（~26.5%），只是遠不及 schedule 的主導地位。
